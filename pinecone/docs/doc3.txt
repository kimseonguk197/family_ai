Python 비동기 프로그래밍과 LangGraph 상태 관리
비동기 I/O 처리는 현대 백엔드 시스템의 성능을 결정짓는 요소입니다.
파이썬의 asyncio 라이브러리는 이를 가능하게 하는 핵심 도구입니다.
에이전트 시스템에서 발생하는 수많은 API 호출을 효율적으로 처리합니다.
LangGraph는 이러한 비동기 환경에서 복잡한 로직을 그래프로 관리합니다.
상태 관리의 핵심은 각 노드 간의 데이터 전달과 유지에 있습니다.
체크포인트 기능은 그래프의 실행 상태를 영속화하는 중요한 역할을 합니다.
PostgreSQL과 같은 DB를 백엔드로 사용하여 상태를 저장합니다.
이를 통해 긴 실행 시간이 걸리는 워크플로우도 안전하게 관리됩니다.
사용자의 승인이 필요한 단계에서 실행을 멈추고 상태를 보존할 수 있습니다.
이후 사용자가 다시 실행을 요청하면 중단된 지점부터 재개됩니다.
이것이 바로 Human-in-the-loop 패턴의 핵심적인 구현 방식입니다.
LangGraph의 상태는 TypedDict를 사용하여 정의되는 것이 일반적입니다.
각 노드는 상태를 인자로 받아 수정된 상태의 일부를 반환합니다.
반환된 데이터는 리듀서 함수에 의해 기존 상태와 병합됩니다.
불변성을 유지하면서 상태를 업데이트하는 것이 설계의 포인트입니다.
에러가 발생했을 때 이전 체크포인트로 돌아가는 복구 기능도 제공합니다.
비동기 노드 함수를 정의할 때는 async def 키워드를 사용합니다.
await 문을 통해 네트워크 요청이나 DB 작업의 완료를 기다립니다.
멀티 스레딩과 달리 컨텍스트 스위칭 비용이 적어 효율적입니다.
